1 系统概述

Identity Center 是一个基于 OAuth 2.0 / OIDC 协议的统一身份认证中心，为所有业务应用提供：

  • ✓ 统一用户管理：用户只需一个账号即可访问所有接入系统
  • ✓ 单点登录 (SSO)：一次登录，多处通行
  • ✓ 多租户支持：支持企业级多租户隔离
  • ✓ 第三方登录：支持 GitHub、Google、微信等第三方账号登录
  • ✓ 权限管理：统一的角色和权限控制

步骤 1：引导用户授权

将用户重定向到授权端点：

GET /api/auth/authorize?client_id={client_id}&redirect_uri={redirect_uri}&response_type=code&scope=openid profile email&state={state}
参数 必填 说明
client_id是应用唯一标识
redirect_uri是授权回调地址，必须与注册时一致
response_type是固定值 code
scope否权限范围，默认 openid profile email
state推荐防 CSRF 攻击的随机字符串

步骤 2：接收授权码

用户授权后，系统重定向回您的回调地址：

GET https://your-app.com/callback?code={authorization_code}&state={state}

⚠️ 请验证返回的 state 是否与发送的一致，防止 CSRF 攻击

步骤 3：换取 Access Token

在后端服务中使用授权码换取 Token：

POST /api/auth/token
Content-Type: application/json

{
  "grant_type": "authorization_code",
  "client_id": "your_client_id",
  "client_secret": "your_client_secret",
  "code": "authorization_code",
  "redirect_uri": "https://your-app.com/callback"
}

响应示例：

{
  "accessToken": "at_01b0df5d-850d-4637-a03f-4f262da208d1",
  "refreshToken": "rt_02c1eg6e-961e-5748-b14g-5g373eb319e2",
  "expiresIn": 3600,
  "tokenType": "Bearer"
}

6 接入示例代码

Node.js / Express
// config.js - 配置信息
const config = {
  IDC_BASE_URL: window.IDC_CONFIG.baseUrl,
  CLIENT_ID: 'your_client_id',
  CLIENT_SECRET: 'your_client_secret',
  REDIRECT_URI: 'https://your-app.com/callback',
};

// 登录入口
router.get('/login', (req, res) => {
  const state = crypto.randomBytes(16).toString('hex');
  req.session.oauth_state = state;

  const authUrl = `${config.IDC_BASE_URL}/api/auth/authorize?` +
    `client_id=${config.CLIENT_ID}&` +
    `redirect_uri=${encodeURIComponent(config.REDIRECT_URI)}&` +
    `response_type=code&` +
    `scope=openid profile email&` +
    `state=${state}`;

  res.redirect(authUrl);
});

// OAuth 回调
router.get('/callback', async (req, res) => {
  const { code, state } = req.query;

  // 验证 state
  if (state !== req.session.oauth_state) {
    return res.status(400).send('Invalid state');
  }

  // 换取 Token
  const tokenRes = await fetch(`${config.IDC_BASE_URL}/api/auth/token`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      grant_type: 'authorization_code',
      client_id: config.CLIENT_ID,
      client_secret: config.CLIENT_SECRET,
      code,
      redirect_uri: config.REDIRECT_URI,
    }),
  });
  
  const tokens = await tokenRes.json();

  // 获取用户信息
  const userRes = await fetch(`${config.IDC_BASE_URL}/api/auth/userinfo`, {
    headers: { Authorization: `Bearer ${tokens.accessToken}` },
  });
  
  const user = await userRes.json();
  
  // 创建本地会话
  req.session.user = user;
  res.redirect('/dashboard');
});
Python / Flask
import secrets
import requests
from flask import Blueprint, request, redirect, session

auth_bp = Blueprint('auth', __name__)

IDC_BASE_URL = 'YOUR_IDC_BASE_URL'  # 从页面配置获取
CLIENT_ID = 'your_client_id'
CLIENT_SECRET = 'your_client_secret'
REDIRECT_URI = 'https://your-app.com/callback'

@auth_bp.route('/login')
def login():
    state = secrets.token_hex(16)
    session['oauth_state'] = state
    
    auth_url = f"{IDC_BASE_URL}/api/auth/authorize?" + \
               f"client_id={CLIENT_ID}&" + \
               f"redirect_uri={REDIRECT_URI}&" + \
               f"response_type=code&" + \
               f"scope=openid profile email&" + \
               f"state={state}"
    
    return redirect(auth_url)

@auth_bp.route('/callback')
def callback():
    code = request.args.get('code')
    state = request.args.get('state')
    
    if state != session.get('oauth_state'):
        return 'Invalid state', 400
    
    # 换取 Token
    token_res = requests.post(f"{IDC_BASE_URL}/api/auth/token", json={
        'grant_type': 'authorization_code',
        'client_id': CLIENT_ID,
        'client_secret': CLIENT_SECRET,
        'code': code,
        'redirect_uri': REDIRECT_URI,
    })
    
    tokens = token_res.json()
    
    # 获取用户信息
    user_res = requests.get(
        f"{IDC_BASE_URL}/api/auth/userinfo",
        headers={'Authorization': f"Bearer {tokens['accessToken']}"}
    )
    
    user = user_res.json()
    session['user'] = user
    
    return redirect('/dashboard')

7 Token 管理

Token 类型 有效期 说明
Authorization Code5 分钟一次性使用，用后即废
Access Token1 小时用于访问 API
Refresh Token7 天用于刷新 Access Token

刷新 Token

当 Access Token 过期时，使用 Refresh Token 获取新的 Token：

POST /api/auth/token
Content-Type: application/json

{
  "grant_type": "refresh_token",
  "refresh_token": "your_refresh_token"
}

8 常见问题

Q: 授权码只能使用一次吗？

是的，授权码使用后会立即失效。如果重复使用会返回错误。

Q: Access Token 过期后怎么办？

使用 Refresh Token 刷新获取新的 Access Token。如果 Refresh Token 也过期了，需要用户重新登录。

Q: 如何实现单点登录？

用户在第一个应用登录后，Identity Center 会保持登录状态。当用户访问第二个应用时，直接完成授权，无需再次输入密码。

Q: Client Secret 忘记了怎么办？

在管理后台"应用管理"页面，点击应用旁边的"查看密钥"按钮即可随时查看 Client Secret。如果需要重置密钥，可以点击"重新生成密钥"按钮。

Q: API 返回 401 错误怎么办？

检查：1) Access Token 是否正确传递；2) Token 是否已过期；3) Token 是否已被撤销。
